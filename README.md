### Common Issues ‡πÅ‡∏•‡∏∞‡∏ß‡∏¥‡∏ò‡∏µ‡πÅ‡∏Å‡πâ‡πÑ‡∏Ç

#### 1. Flutter Doctor Issues

**Problem:** ‚ùå Android toolchain not found
```bash

```

**Problem:** ‚ùå Android Studio not found
```bash

```

**Problem:** ‚ùå Command line tools not available
```bash

# Open Android Studio ‚Üí Preferences ‚Üí Appearance & Behavior 
# ‚Üí System Settings ‚Üí Android SDK ‚Üí SDK Tools
# Check "Android SDK Command-line Tools" ‚Üí Apply
```

#### 2. Build ‡πÅ‡∏•‡∏∞ Run Issues

**Problem:** Build fails with "Gradle task assembleDebug failed"
```bash
# Solution 1: Clean and rebuild
flutter clean
flutter pub get
flutter run

# Solution 2: Check internet connection
# Gradle needs to download dependencies

# Solution 3: Update Gradle wrapper
cd android
./gradlew wrapper --gradle-version=8.0
cd ..
```

**Problem:** "Waiting for another flutter command to release the startup lock"
```bash
# Solution: Remove the lock file
rm -rf ~/.flutter-devtools/.pid
# or
rm -rf $FLUTTER_ROOT/bin/cache/lockfile
```

**Problem:** Hot reload not working
```bash
# Solution 1: Try hot restart (press 'R')
# Solution 2: Stop and restart app completely
# Solution 3: Clean project
flutter clean
flutter pub get
flutter run
```

#### 3. Emulator/Device Issues

**Problem:** No devices found
```bash
# Check devices
flutter devices

# For Android Emulator:
flutter emulators
flutter emulators --launch <emulator_id>

# For iOS Simulator (macOS):
open -a Simulator

# For physical device:
# - Enable Developer Mode
# - Enable USB Debugging
# - Accept computer authorization
```

**Problem:** Emulator very slow
```bash
# Solution 1: Allocate more RAM
# Android Studio ‚Üí AVD Manager ‚Üí Edit ‚Üí Advanced Settings ‚Üí RAM

# Solution 2: Enable Hardware Acceleration
# Settings ‚Üí Enable Intel VT-x or AMD-V in BIOS

# Solution 3: Use a simpler device
# Create emulator with lower API level (API 28-30)
```

#### 4. Dependencies Issues

**Problem:** "Package not found" error
```bash
# Solution:
flutter clean
rm pubspec.lock
flutter pub get

# If still fails:
flutter pub cache repair
```

**Problem:** Version conflict errors
```bash
# Solution:
# Check pubspec.yaml for dependency conflicts
# Run:
flutter pub upgrade
flutter pub get
```

#### 5. IDE Issues

**Problem:** VS Code not recognizing Flutter
```bash
# Solution:
# 1. Install Flutter extension
# 2. Reload VS Code (Cmd/Ctrl + Shift + P ‚Üí "Reload Window")
# 3. Check Flutter SDK path (Cmd/Ctrl + Shift + P ‚Üí "Flutter: Change SDK")
```

**Problem:** Android Studio Flutter plugin issues
```bash
# Solution:
# 1. File ‚Üí Settings ‚Üí Plugins
# 2. Uninstall Flutter and Dart plugins
# 3. Restart Android Studio
# 4. Reinstall plugins
# 5. Restart again
```

#### 6. Platform-Specific Issues

**macOS - Xcode Issues:**
```bash
# Problem: Xcode version too old
# Solution: Update Xcode from App Store

# Problem: CocoaPods not found

# Problem: Signing issues
# Solution: Open Xcode ‚Üí Project ‚Üí Signing & Capabilities

```

**Windows - Path Issues:**
```bash

# Solution: Add Flutter to PATH
# 1. Search "Environment Variables"
# 2. Edit PATH variable
# 3. Add C:\src\flutter\bin (or your Flutter location)
# 4. Restart Command Prompt
```

**Linux - Permission Issues:**
```bash

# Solution:

# Problem: USB device not recognized
# Solution:

# Then logout and login again
```

#### 7. Performance Issues

**Problem:** App runs slowly
```bash
# Solution 1: Use release mode

# Solution 2: Profile the app

# Then use DevTools to identify bottlenecks

# Solution 3: Check device specifications

```

**Problem:** App size too large
```bash

# Check what's taking space:

```

#### 8. Error Messages Reference

**"A RenderFlex overflowed"**
```dart
// Problem: Widget doesn't fit in available space
// Solution: Wrap with Expanded or Flexible
Expanded(
  child: YourWidget(),
)
```

**"setState() called after dispose()"**
```dart
// Problem: Calling setState on disposed widget
// Solution: Check if mounted before setState
if (mounted) {
  setState(() {
    // your state changes
  });
}
```

**"Vertical viewport was given unbounded height"**
```dart
// Problem: ListView inside Column without height constraint
// Solution: Wrap ListView with Expanded
Column(
  children: [
    Expanded(
      child: ListView(...),
    ),
  ],
)
```
